# 手工数据采集平台

## 平台主要做什么
手工数据采集平台用于把业务侧 Excel/CSV 数据，按统一流程导入到 MySQL 目标表。

核心流程：
- 导入方案管理：新建、编辑、启停、删除方案
- 模板导出：按目标表结构导出模板
- 数据导入：上传文件、解析 Sheet、字段映射（手工/自动）
- 数据校验：必填校验、类型校验、错误导出
- 数据入库：支持 APPEND / UPSERT / PARTITION_OVERWRITE / FULL_OVERWRITE
- 批次回滚：支持按写入模式差异化回滚
- 任务管理：任务列表、详情、日志、单删与批量删除

## 能带来什么价值
- 提升数据质量：入库前校验和错误阻断，减少脏数据进入生产表
- 提高导入效率：模板导出 + 自动映射，减少人工整理时间
- 降低运维风险：批次可追踪、可回滚，出现问题可快速止损
- 过程可审计：任务、批次、日志全链路留痕，便于复盘和追责
- 支持规模化协同：统一入口与规范流程，减少“个人脚本式”导入

## 技术架构
- 前端：React + Vite + Ant Design
- 后端：Node.js + Express + TypeScript
- 数据库：MySQL

## 最新能力（2026-05-22）
- 审批流升级为流程引擎：支持串行（SERIAL）、或签（OR_SIGN）、会签（AND_SIGN）。
- 支持通过阈值与驳回阈值：可配置最少通过人数（MIN_PASS_COUNT）和阈值驳回（THRESHOLD_REJECT）。
- 新增审批流模板管理：支持模板创建、编辑、发布，支持节点审批人按用户/角色/域管理员配置。
- 新增审批中心：支持我的待办、我的申请、审批通过/驳回。
- 提交入库与审批闭环联动：未达到审批通过条件前阻断最终写入。
- 双库隔离：元数据与系统表使用 `DB_NAME`，业务目标表访问使用 `TARGET_DB_NAME`。

## 快速启动

### 1. 启动后端
```bash
cd backend
npm install
npm run dev
```

### 2. 启动前端
```bash
cd frontend
npm install
npm run dev
```

- 前端：http://localhost:5174
- 后端：http://localhost:3001

## 说明
请勿将真实数据库密码、密钥等敏感信息提交到仓库。
使用 `backend/.env.example` 复制为本地 `.env` 后再配置。

## 双库配置建议
生产环境建议将元数据与业务目标数据分离：
- `DB_NAME`：元数据库（用户、权限、审批、任务、日志等系统表）
- `TARGET_DB_NAME`：业务目标库（实际同步的业务目标表）
- `META_DB_NAME`：回滚快照与辅助元数据（通常与 `DB_NAME` 一致）

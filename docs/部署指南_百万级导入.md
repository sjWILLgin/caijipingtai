# 手工数据采集平台部署指南（百万级导入）

## 1. 文档目标
本指南用于生产部署手工数据采集平台，按默认单次上传 100 万行数据场景给出环境、规格、部署步骤、运维建议。

## 2. 版本与环境要求

### 2.1 操作系统
- Ubuntu 22.04 LTS（推荐）
- 兼容：Ubuntu 20.04、Rocky Linux 8+
- 兼容：CentOS 7/8（建议优先 CentOS Stream 8 或 Rocky 8/9）

### 2.2 运行时
- Node.js 20 LTS
- npm 10+
- MySQL 8.0（推荐）或 5.7（兼容）
- Nginx 1.20+
- PM2 5+

### 2.3 基础工具
- git
- curl
- unzip
- lsof
- python3（可选，排障辅助）

### 2.4 CentOS 快速安装（一次性）

CentOS 8 / Stream 8 / Rocky 8：

```bash
sudo dnf -y update
sudo dnf -y install epel-release git curl unzip lsof python3 nginx

# Node.js 20
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo dnf -y install nodejs

# PM2
sudo npm i -g pm2

# MySQL 8（使用社区仓库）
sudo dnf -y install https://dev.mysql.com/get/mysql80-community-release-el8-1.noarch.rpm
sudo dnf -y install mysql-community-server
sudo systemctl enable --now mysqld

# Nginx
sudo systemctl enable --now nginx
```

CentOS 7：

```bash
sudo yum -y update
sudo yum -y install epel-release git curl unzip lsof python3 nginx

# Node.js 20
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum -y install nodejs

# PM2
sudo npm i -g pm2

# MySQL 8
sudo yum -y install https://dev.mysql.com/get/mysql80-community-release-el7-7.noarch.rpm
sudo yum -y install mysql-community-server
sudo systemctl enable --now mysqld

# Nginx
sudo systemctl enable --now nginx
```

安装后建议检查：

```bash
node -v
npm -v
pm2 -v
nginx -v
mysql --version
```

## 3. 推荐规格（默认百万行上传）

### 3.1 单机最小可用（测试）
- CPU：4 vCPU
- 内存：8 GB
- 磁盘：100 GB SSD
- MySQL：同机部署

### 3.2 生产推荐（中小团队）
- 应用机：8 vCPU / 16 GB / 200 GB SSD
- 数据库机：8 vCPU / 32 GB / 500 GB SSD
- 网络：千兆内网

### 3.3 稳定型（并发导入）
- 应用机：16 vCPU / 32 GB
- 数据库机：16 vCPU / 64 GB
- MySQL 独立实例 + 主从备份

## 4. 目录建议
- 代码目录：/opt/data-collection
- 上传目录：/data/data-collection/uploads
- 日志目录：/var/log/data-collection
- 备份目录：/data/backup/data-collection

## 5. 数据库与账号体系说明

### 5.1 数据库分层
当前系统使用两个数据库：
- data_collection_platform：业务数据与平台元数据
- data_collection_meta：回滚快照与元仓类辅助数据

### 5.2 账号密码存储
- 账号密码存储于数据表 sys_user
- 密码字段为 password_hash（bcrypt 哈希）
- 不存明文密码

### 5.3 内置超管账号
系统启动时会确保存在：
- 用户名：root
- 密码：zaowang123
- 角色：super_admin（超级管理员）

> 建议首次上线后立即修改 root 密码。

## 6. 后端部署

### 6.1 配置环境变量
在 backend 目录创建 .env：

```env
PORT=3001
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=请替换为生产密码
DB_NAME=data_collection_platform
META_DB_NAME=data_collection_meta
JWT_SECRET=请替换为高强度随机串
JWT_EXPIRES_IN=12h
IMPORT_CHUNK_SIZE=2000
IMPORT_MAX_ROWS=1200000
IMPORT_MAX_TARGET_ROWS=10000000
ROLLBACK_SNAPSHOT_MAX_ROWS=10000000
```

### 6.2 安装与构建
```bash
cd /opt/data-collection/backend
npm ci
npm run build
```

### 6.3 启动（PM2）
```bash
pm2 start dist/index.js --name data-collection-backend
pm2 save
pm2 startup
```

## 7. 前端部署

### 7.1 构建
```bash
cd /opt/data-collection/frontend
npm ci
npm run build
```

### 7.2 Nginx 托管 dist
将 frontend/dist 发布到 Nginx 站点目录，例如 /var/www/data-collection。

## 8. Nginx 示例配置

```nginx
server {
  listen 80;
  server_name your-domain.com;

  root /var/www/data-collection;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location /api/ {
    proxy_pass http://127.0.0.1:3001/api/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 300s;
  }

  location /uploads/ {
    proxy_pass http://127.0.0.1:3001/uploads/;
    proxy_set_header Host $host;
  }
}
```

## 9. 百万级上传参数建议

### 9.1 后端参数
- IMPORT_CHUNK_SIZE=2000
- IMPORT_MAX_ROWS=1200000
- IMPORT_MAX_TARGET_ROWS=10000000

### 9.2 MySQL建议
- innodb_buffer_pool_size：物理内存 50%-70%
- max_allowed_packet：64M
- innodb_log_file_size：512M
- 开启慢查询日志（long_query_time=1）

### 9.3 应用建议
- 上传文件限制建议 150MB 以内
- 使用异步队列提交（系统已支持）
- 大批次优先在业务低峰时段执行

## 10. 健康检查与验证

### 10.1 接口健康
```bash
curl http://127.0.0.1:3001/api/health
```

### 10.2 登录检查
- 打开前端 /login
- 使用 root / zaowang123 登录
- 可见“用户权限”菜单

### 10.3 权限检查
- 新注册账号默认为 analyst
- analyst 不显示“用户权限”菜单
- 超管可在“用户权限”页配置勾选权限

## 11. 安全建议
- 强制 HTTPS
- JWT_SECRET 不可使用默认值
- 仅开放 80/443，3001 仅内网
- MySQL 仅内网访问
- 定期备份 data_collection_platform 与 data_collection_meta

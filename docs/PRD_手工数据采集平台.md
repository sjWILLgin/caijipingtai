# 手工数据采集平台 PRD

## 1. 文档信息
- 产品名称：手工数据采集平台
- 文档版本：v1.1
- 更新时间：2026-05-21
- 适用对象：业务用户、产品、开发、测试、运维

## 2. 背景与目标
### 2.1 背景
业务侧存在大量 Excel/CSV 手工数据导入场景，当前导入方式分散、缺乏统一校验、过程不可审计，且回滚成本高。

### 2.2 目标
- 建立标准化导入流程：方案 -> 上传 -> Sheet配置 -> 字段映射 -> 校验 -> 入库 -> 回滚。
- 降低错误入库概率，支持阻断校验与错误导出。
- 建立任务级和批次级审计追踪，支持回滚和复盘。
- 支持模板导出、自动映射、批量任务管理。
- 支持大数据量导入的稳态处理（分块、限流、异步作业化）。
- 将回滚升级为“数据状态恢复”而非简单清空。

### 2.3 非目标
- 不覆盖实时流式采集。
- 不做复杂审批流（当前仅保留可扩展位）。

## 3. 用户角色
- 业务操作员：创建方案、发起导入、查看错误、提交入库。
- 数据管理员：维护规则、处理失败任务、执行回滚。
- 开发/运维：维护服务、排障、扩展能力。

## 4. 核心业务流程
1. 创建/编辑导入方案。
2. 选择目标表并导出模板。
3. 发起导入任务并上传文件。
4. 配置 Sheet 导入策略。
5. 字段映射（手工或自动）。
6. 执行校验并处理错误。
7. 提交入库（APPEND/FULL_OVERWRITE/PARTITION_OVERWRITE/UPSERT）。
8. 在任务详情查看批次、日志，必要时回滚。

## 5. 功能需求
### 5.1 导入方案管理
- 新建、编辑、启用、停用、删除方案。
- 方案版本化（编辑后生成新版本）。
- 配置项：业务域、数据主题、文件类型、Sheet策略、目标表、写入方式、说明。
- 约束：方案被任务引用时不可删除。

### 5.2 模板导出
- 在新建/编辑方案页选择目标表后，支持导出该表模板（CSV表头）。
- 模板字段来自数据库元数据，默认排除自增主键。

### 5.3 任务管理
- 按方案创建任务。
- 任务列表支持筛选、查看详情、单条删除、批量删除。
- 删除任务时执行级联清理（任务、映射、校验、批次、日志、文件记录、目标表数据）。

### 5.4 文件上传与解析
- 支持 xlsx/xls/csv。
- 上传后通过异步作业队列执行解析，生成初始 Sheet 与字段映射。
- 文件大小限制：50MB（可配置）。

新增要求：
- 文件 hash 使用流式计算，避免整文件读取导致内存峰值过高。
- 解析需支持大文件分块处理与总行数上限保护。

### 5.5 Sheet 配置
- 每个 Sheet 可设置是否导入、目标表、是否有表头、表头行、数据起始行。
- 校验规则：数据起始行必须大于表头行（有表头时）。

### 5.6 字段映射
- 手工映射：逐字段选择目标字段。
- 自动映射：支持两种策略
  - SAME_NAME（同名映射）
  - ORDER（同行映射）
- 保存映射后必须清理历史校验结果，并要求重新校验。

### 5.7 数据校验
- 校验维度：
  - 必填字段未映射
  - 必填字段为空
  - 目标字段类型不匹配（数值/日期等）
- 结果分级：BLOCKING / WARNING。
- 输出：错误明细、阻断数、警告数、成功行数、规则数量。
- 支持错误导出。

新增要求：
- 校验通过异步作业队列执行。
- 校验按块处理，避免整表加载到内存。
- 超过上限行数时给出可理解错误并引导分批导入。

### 5.8 入库与回滚
- 仅 READY/VALIDATE_SUCCESS 可提交入库。
- 入库结果需可追踪到 batch_id。
- 入库通过异步作业队列执行。
- 回滚策略升级：
  - APPEND：删除指定 batch_id 数据。
  - FULL_OVERWRITE / PARTITION_OVERWRITE / UPSERT：优先从元仓快照恢复提交前数据状态。
  - 若无快照或恢复失败，再回退到兼容策略。
- 支持对“已失效批次”补偿清理，避免历史残留数据。

新增要求：
- 引入元仓库（data_collection_meta）保存回滚快照元数据与快照表。
- 回滚结果需记录快照恢复状态（CREATED/RESTORED/FAILED）。

### 5.9 日志与审计
- 全流程操作记录审计日志。
- 任务详情展示日志时间线。
- 关键动作（上传、解析、校验、入库、回滚）必须可追溯。

新增要求：
- 增加表级留痕：支持按表查看批次、操作者、方案版本、文件信息、映射快照与脱敏样例。
- 支持留痕筛选与导出（CSV/JSON）。

### 5.10 异步作业中心
- 新增 async_job 作业模型，覆盖 PARSE / VALIDATE / COMMIT。
- 新增接口：
  - GET /api/jobs
  - GET /api/jobs/:jobId
- 接口触发后返回 job_id，前端可轮询状态。

## 6. 状态机
- 任务状态：
  - DRAFT
  - PARSING / PARSE_SUCCESS / PARSE_FAILED
  - MAPPING
  - VALIDATING / READY / VALIDATE_FAILED
  - COMMITTING / SUCCESS / COMMIT_FAILED
  - ROLLED_BACK / CANCELLED

## 7. 非功能需求
- 可用性：核心页面响应时间 < 3s（不含大文件上传）。
- 性能：支持万行级稳定导入，默认上限 200000 行（可配置）。
- 安全：表名白名单校验、防SQL注入、文件类型限制。
- 可维护：模块化路由与服务层分离。

新增指标：
- 内存稳态：禁止在上传 hash、解析、校验、入库阶段进行整表常驻内存处理。
- 可观测性：异步作业需有状态、耗时、失败原因。

## 8. 验收标准
- 方案/任务/上传/映射/校验/入库/回滚全链路可执行。
- 错误数据被阻断，不会误入库。
- 模板可在方案阶段导出。
- 任务支持单删与批删。
- 回滚后目标表数据符合预期。
- 大数据导入时系统可持续响应，无明显卡死或内存异常。
- PARSE/VALIDATE/COMMIT 均可返回 job_id 并查询状态。
- FULL_OVERWRITE 场景可恢复到提交前数据状态（快照恢复）。
- 表级留痕可按条件检索并导出。

## 9. 后续规划

- 更细粒度权限模型。

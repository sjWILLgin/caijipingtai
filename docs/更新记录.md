# 采集平台更新记录

## 记录规则
- 每次功能变更后，同步更新 docs 下对应文档。
- 本文件仅记录“做了什么、影响什么、对应文档”。

## 2026-05-22

### 账号与权限
- 新增注册/登录/鉴权流程。
- 新增固定超管账号 root。
- 调整为仅 root 可担任 super_admin（其余账号为 analyst）。
- 新增用户权限矩阵（按用户勾选）。
- 新增修改密码功能。
- 新增接口级权限拦截。
- 新增超管重置他人密码功能。
- 新增超管删除账号功能（root 与当前账号不可删除）。
- 新增删除账号审计日志（写入 audit_log）。
- 顶部操作改为账户下拉（修改密码/退出登录）。
- 新增超管信息中心（查看全员当日操作记录，支持筛选）。
- 新增域管理员角色（domain_admin）与域绑定能力。
- 新增用户权限页“域绑定”可视化弹窗（超管可直接维护账号业务域）。
- 新增手工数据表审批流配置（按表配置是否强制审批、审批角色与审批人）。
- 新增提交入库审批闭环（触发审批时阻断最终写入，审批通过后方可提交）。
- 新增审批中心页面（我的待办/我的申请，支持通过与驳回）。
- 审批流升级为可编排流程引擎：支持 SERIAL / OR_SIGN / AND_SIGN。
- 新增审批流模板管理页面与 API（创建/编辑/发布/查看详情）。
- 新增节点审批人配置模型（USER / ROLE / DOMAIN_ADMIN）。
- 新增通过阈值与驳回阈值能力：
	- 通过策略：MIN_PASS_COUNT
	- 驳回策略：THRESHOLD_REJECT
- 提交入库与审批流实例联动：审批未完成时阻断最终写入，满足通过条件后放行。

### 数据库分层与目标表隔离
- 新增 `TARGET_DB_NAME` 环境变量，用于业务目标表读写。
- 目标表访问链路按 `TARGET_DB_NAME` 执行：
	- 提交入库链路（schema 查询、写入、覆盖更新）
	- 校验链路（目标字段类型读取）
	- 映射链路（目标字段发现）
- 手工数据表接口强化隔离：
	- `/api/tables` 与 `/api/tables/manual/overview` 仅返回业务目标表
	- 过滤系统元数据表，避免元表暴露到用户侧
- 回滚快照仍存放于元数据域（`META_DB_NAME`），与业务目标库分离。

### 审批模板命中规则与导入联动（新增）
- 审批流模板新增“绑定目标表”配置（一个模板可绑定多个目标表）。
- 导入方案“需要审批”改为规则驱动：
	- 命中模板时自动为“需要审批”，且前端不可手动关闭。
	- 未命中模板时自动为“不需要审批”，前端置灰不可手动开启。
- 导入方案页新增审批链预览：命中模板后可直接查看模板、节点、签署规则。
- 提交确认页新增命中审批流展示，提交前可看到将进入的审批节点。
- 提交入库后端改为按目标表命中模板判定审批，不再依赖用户手工开关。
- 新增审批模板命中查询接口：`GET /api/import-plans/approval-rule`。

### 审批模板编辑体验优化（新增）
- `flow_code` 前后端统一格式校验，前端实时转大写并过滤非法字符。
- 节点审批人配置支持下拉选择：
	- 指定用户：下拉注册用户列表（不再手填用户ID）
	- 角色：下拉角色列表（不再手填角色键）
- 手工数据表审批配置改为模板驱动审批状态：选中模板自动视为强制审批。

### 审批展示文案简化（新增）
- 导入方案页与提交确认页的审批链展示改为业务可读文案，不再直接展示技术术语。
- 展示语义统一为“审批方式、通过条件、终止条件、审批人”，便于业务同学快速理解。
- 常见展示示例：
	- 审批方式：按顺序逐级审批 / 任意一人同意即可 / 需要多人共同同意
	- 通过条件：至少 N 人同意 / 全部同意才通过
	- 终止条件：有人驳回即终止 / 达到 N 人驳回即终止

### 模板绑定回显修复（新增）
- 修复“审批流模板已绑定目标表，但返回后再次进入显示为空”的问题。
- 修复“导入方案侧审批规则未随模板绑定生效”的问题。
- 兼容策略：当模板表中的目标表字段为空时，自动从元仓表 `manual_table_approval_config` 回填读取已绑定关系。
- 已对现网历史数据执行一次性回填，恢复已有绑定关系可见性。

### 审批模板删除能力（新增）
- 审批流模板管理页新增“删除模板”按钮。
- 删除时会自动解除该模板与手工数据表的绑定关系（审批置为关闭）。
- 若模板已有审批实例记录，则不允许删除并给出提示（避免历史审批数据断链）。

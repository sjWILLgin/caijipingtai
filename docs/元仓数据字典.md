# 数据采集平台 - 元仓数据字典

## 文档说明
本文档详细记录了**数据采集平台**的元数据仓库（Meta Repository）结构、各表用途、字段定义及常用查询。
- **元数据库**: `data_collection_platform`
- **目标数据库**: `data_collection_target`
- **快照数据库**: `data_collection_meta`
- **更新时间**: 2026-05-25
- **说明**: 生产环境使用阿里云 RDS，系统表与业务目标表按库隔离。

## 📋 表结构详解

### 1️⃣ **async_job** - 异步任务队列表
**用途**: 记录异步处理任务（解析、校验、提交）的执行状态和进度
**行数**: 1

#### 字段定义

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int(11) | PRIMARY | 自增主键 |
| job_id | varchar(50) | UNIQUE | 异步任务唯一标识 |
| task_id | varchar(50) | FOREIGN | 关联导入任务ID |
| job_type | enum('PARSE','VALIDATE','COMMIT') | NOT NULL | 任务类型：解析/校验/提交 |
| status | enum('PENDING','RUNNING','SUCCESS','FAILED') | - | 任务状态 |
| progress | int(11) | - | 进度百分比 (0-100) |
| result | json | - | 任务结果（成功时存储） |
| error_message | text | - | 错误信息（失败时存储） |
| started_at | datetime | - | 任务开始时间 |
| finished_at | datetime | - | 任务完成时间 |
| created_at | datetime | - | 记录创建时间 |

#### 关键业务逻辑
- 单个task可能对应多个job（PARSE → VALIDATE → COMMIT 三个阶段）
- status 字段追踪任务生命周期
- progress 用于UI展示，completed = progress >= 100

## 🔍 常用查询

### 1. 查看最近7天的任务统计
```sql
SELECT 
    DATE(created_at) as date,
    COUNT(DISTINCT task_id) as total_tasks,
    SUM(CASE WHEN status = 'SUCCESS' THEN 1 ELSE 0 END) as success_count,
    SUM(CASE WHEN status = 'FAILED' THEN 1 ELSE 0 END) as failed_count
FROM import_task
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY DATE(created_at)
ORDER BY date DESC;
```

### 2. 统计每个数据集（任务）的导入成功率
```sql
SELECT 
    it.task_id,
    it.creator_name,
    COUNT(ib.batch_id) as batch_count,
    SUM(ib.success_count) as total_success,
    SUM(ib.total_count) as total_records,
    ROUND(100.0 * SUM(ib.success_count) / SUM(ib.total_count), 2) as success_rate
FROM import_task it
LEFT JOIN import_batch ib ON it.task_id = ib.task_id
WHERE it.created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
```

## 📈 关键指标计算公式

### 导入成功率
```
成功率 = (成功记录数 / 总记录数) × 100%
来源: import_batch.success_count / import_batch.total_count
```

### 任务阻塞率
```
阻塞率 = (阻塞性错误数 > 0 的任务 / 总任务数) × 100%
来源: WHERE blocking_error_count > 0
```

### 任务完成时间
```
完成时间 = finished_at - created_at (异步任务)
完成时间 = updated_at - created_at (导入任务)
```

### 数据质量指标
```
数据质量 = (is_valid = 1 的记录 / 总记录数) × 100%
```

### 方案使用频率
```
使用频率 = 方案关联的任务数 / 时间周期
```

## 📝 常见问题排查

### Q1: 为什么某个任务显示为"进行中"但很久没更新？
**答**: 检查 `async_job` 表中该任务对应的 job 状态。如果为 'RUNNING'，检查后端日志；如果状态异常，可能需要重启任务。

### Q2: 如何找到某次导入失败的原因？
**答**: 
1. 通过 `import_task.task_id` 查询 `audit_log`（log_level='ERROR'）
2. 查询 `validate_error` 表的具体错误信息
3. 检查 `async_job.error_message` 中的错误堆栈

### Q3: 如何回滚数据？
**答**: 
1. APPEND 模式按 `batch_id` 删除目标库新增数据。
2. FULL_OVERWRITE / PARTITION_OVERWRITE / UPSERT 优先从 `META_DB_NAME` 中的快照表恢复提交前状态。
3. 回滚后检查 `import_batch` 的 `is_valid`、`is_latest` 与审计日志。

### Q4: 字段映射如何重用？
**答**: 在新任务的字段映射中设置 `mapping_type='LAST_MAPPING'`，系统会自动查询上次使用该方案的映射。

### Q5: 如何查询某业务域的数据导入历史？
**答**: 
```sql
SELECT it.* FROM import_task it
JOIN import_plan ip ON it.plan_id = ip.plan_id
WHERE ip.domain = '销售域'
ORDER BY it.created_at DESC;
```

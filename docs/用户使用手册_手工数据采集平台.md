# 手工数据采集平台 用户使用手册

## 1. 适用对象
面向业务同学和数据运营同学，用于日常导入、校验、入库、回滚。

## 2. 快速上手
1. 打开系统首页。
2. 进入“导入方案”创建或选择方案。
3. 选择目标表后先导出模板。
4. 按模板整理文件并上传。
5. 完成 Sheet 配置、字段映射。
6. 执行校验，修复错误后重新校验。
7. 校验通过后提交入库。
8. 在任务详情查看批次与日志，必要时回滚。

## 3. 详细操作

## 3.0 审批模板维护（管理员）
路径：系统管理 -> 审批流模板

适用角色：super_admin、domain_admin

可执行操作：
- 新建模板：配置模板名称、适用域、启用状态
- 配置节点：选择串行/或签/会签策略，设置通过阈值与驳回阈值
- 配置审批人：按指定用户/角色/域管理员维护
- 发布模板：发布后可在手工数据表审批配置中绑定

注意：
- 强制审批的手工数据表必须绑定已发布模板。
- 调整模板后建议新建或重跑任务，避免历史实例语义不一致。

## 3.1 新建导入方案
路径：导入方案 -> 新建方案

必填项建议：
- 方案名称
- 业务域
- 数据主题
- 文件类型（单选）
- Sheet策略
- 目标表
- 入库方式（单选）

注意：
- 选择目标表后可直接点击“导出模板”。
- 方案保存后可启用/停用。
- 方案“是否需要审批”仅由目标表是否命中启用审批模板决定。

## 3.2 导出模板
- 在方案页面选择目标表。
- 点击“导出模板”。
- 系统会下载 CSV 模板（表头即数据库字段）。

建议：
- 先用模板整理数据再上传，可显著降低映射和校验错误。

## 3.3 发起导入任务
路径：导入方案列表 -> 开始导入

系统会自动创建任务并进入上传页面。

## 3.4 上传文件
支持格式：xlsx / xls / csv

注意：
- csv 时请确认编码和分隔符。
- 上传后系统自动解析（异步作业）。
- 返回 `job_id` 后可用于跟踪后台执行状态。

## 3.5 Sheet 配置
- 选择要导入的 Sheet。
- 配置目标表。
- 设置是否有表头、表头行、数据起始行。

常见错误：
- 数据起始行 <= 表头行（会被拦截）。

## 3.6 字段映射
支持三种方式：
- 手工逐字段映射。
- 自动映射（同名映射 SAME_NAME）。
- 自动映射（同行映射 ORDER）。

注意：
- 保存映射后会清理旧校验结果，下一步会重新校验。

## 3.7 校验结果
系统会显示：
- 总行数
- 通过行数
- 阻断错误
- 警告
- 校验规则数量和明细

处理建议：
- 有阻断错误时，先修复再提交。
- 可下载错误文件定位问题。
- 校验为异步执行，可根据任务状态或作业状态确认完成情况。

## 3.8 提交入库
可选入库方式：
- APPEND
- PARTITION_OVERWRITE
- UPSERT
- FULL_OVERWRITE

注意：
- 高风险模式请确认影响范围。
- 提交后可在任务详情跟踪状态，后台会异步执行入库作业。
- 当方案或目标表开启审批时，提交会先进入审批流程，审批通过后才会最终写入目标表。

## 3.11 审批中心（待办/申请）
路径：系统管理 -> 审批中心

功能：
- 我的待办：处理当前待审批事项
- 我的申请：查看自己发起的审批状态
- 审批动作：支持通过/驳回并填写意见

状态说明：
- `PENDING`：流程进行中
- `APPROVED`：达到通过条件，可继续入库
- `REJECTED`：达到驳回条件，入库被拒绝

## 3.12 审批链展示（业务可读）
为方便业务同学理解，页面中的审批链已使用简化文案展示，不需要理解技术术语。

页面位置：
- 导入方案页（命中审批模板时）
- 提交确认页（提交前）

展示维度：
- 审批方式：按顺序逐级审批 / 任意一人同意即可 / 需要多人共同同意
- 通过条件：有人同意即可通过 / 全部同意才通过 / 至少 N 人同意
- 终止条件：有人驳回即终止 / 达到 N 人驳回即终止
- 审批人：由业务域负责人审批 / 由指定角色审批 / 由指定人员审批

使用建议：
- 提交前先看“通过条件”和“终止条件”，确认是否符合当前业务场景。
- 若目标表命中启用审批模板，方案会自动进入审批，不需要手动判断。

## 3.13 手工数据表列表展示规则
路径：手工数据表

当前展示口径：
- 列表直接读取数据库真实业务表。
- 与是否存在历史导入方案无关。
- 初始化清空任务/方案后，已有业务表仍会正常显示。

## 3.9 任务详情与回滚
任务详情可查看：
- 任务基础信息
- 批次信息
- 执行日志

回滚说明：
- APPEND：删除该批次新增数据。
- FULL_OVERWRITE / PARTITION_OVERWRITE / UPSERT：优先恢复至提交前快照状态。
- 无快照时按兼容策略处理。

## 3.10 任务记录删除
- 支持单条删除。
- 支持多选后批量删除。

注意：
- 删除任务会同步清理其映射、校验、批次、日志、文件记录及关联目标表数据。

## 4. 常见问题

### Q1：一直弹“入库成功”怎么办？
- 刷新后仍持续弹，说明前端缓存或页面状态异常，先强刷页面。
- 若仍异常，联系管理员查看该任务日志。

### Q2：回滚后数据还在怎么办？
- 先刷新任务详情确认回滚成功。
- 再查看目标表当前行数。
- 若是历史遗留批次，可执行补偿清理。

### Q3：校验通过但数据不对怎么办？
- 检查映射是否正确。
- 检查字段类型是否匹配。
- 检查使用的是不是正确方案版本。

### Q4：大数据量导入会不会把系统搞挂？
- 平台已启用分块处理 + 异步队列，通常不会因为单次导入直接拖垮服务。
- 当触发限额时会给出明确错误：
	- `ROW_LIMIT_EXCEEDED`：单任务行数超限
	- `TARGET_TABLE_ROW_LIMIT_EXCEEDED`：目标表预计行数超限
	- `SNAPSHOT_TOO_LARGE`：回滚快照过大
- 处理建议：拆分文件分批导入，清理历史数据后再导入。

## 5. 操作建议
- 每次导入前先导出模板。
- 优先使用同名映射，再手工微调。
- 提交前重点看阻断错误和关键字段样例值。
- 大批量导入前先用小样本试跑。
- 大批量导入建议使用 CSV，优先按 1~5 万行每批拆分。
- 高峰期避免并发提交多个超大任务。

# 手工数据采集平台账号与权限矩阵

## 1. 目标
实现最小可用账号体系：
- 注册
- 登录
- 修改密码
- 超管配置用户角色与权限勾选

## 2. 账号体系

### 2.1 用户表
- 表名：sys_user
- 核心字段：username, password_hash, display_name, is_active
- 密码存储：bcrypt 哈希

### 2.2 角色表
- 表名：sys_role
- 角色：
  - super_admin（超级管理员）
  - analyst（分析师）

### 2.3 用户角色关联
- 表名：sys_user_role
- 一个用户当前仅配置一个角色

### 2.4 用户权限关联
- 表名：sys_user_permission
- analyst 通过勾选权限控制可操作范围
- super_admin 默认全权限（不走勾选）

## 5. 权限矩阵

| 模块 | 权限键 | 含义 |
|------|--------|------|
| 导入方案 | plan.view | 查看导入方案 |
| 导入方案 | plan.manage | 新建/编辑/启停/删除导入方案 |
| 任务 | task.view | 查看任务列表与详情 |
| 任务 | task.create | 创建任务、取消任务 |
| 任务 | task.delete | 删除任务 |
| 导入 | file.upload | 上传/下载文件 |
| 导入 | mapping.edit | Sheet配置与字段映射编辑 |
| 导入 | validation.run | 执行校验 |
| 导入 | commit.execute | 提交入库 |
| 导入 | commit.rollback | 回滚批次 |
| 手工数据表 | table.view | 查看表清单、活动、数据 |
| 手工数据表 | table.lifecycle | 修改生命周期策略 |
| 手工数据表 | table.delete | 删除手工数据表 |
| 看板 | dashboard.view | 查看首页统计 |
| 审计 | audit.view | 查看日志与作业状态 |
| 系统管理 | user.manage | 用户与权限管理 |

## 6. 默认权限策略

### 6.1 super_admin
- 默认拥有全部权限
- 可分配角色与勾选他人权限

### 6.2 analyst（默认）
默认勾选：
- plan.view
- task.view
- task.create
- file.upload
- mapping.edit
- validation.run
- commit.execute
- table.view
- dashboard.view
- audit.view

默认不勾选：
- plan.manage
- task.delete
- commit.rollback
- table.lifecycle
- table.delete
- user.manage

## 7. API 权限控制说明
系统按接口路径 + HTTP 方法进行权限拦截。
示例：
- POST /api/commit/:taskId 需要 commit.execute
- POST /api/commit/batches/:batchId/rollback 需要 commit.rollback
- PUT /api/tables/:tableName/lifecycle 需要 table.lifecycle
- DELETE /api/tables/:tableName 需要 table.delete

## 8. 修改密码流程
- 入口：页面头部“修改密码”
- 校验项：旧密码正确、新密码至少 6 位、确认密码一致
- 接口：POST /api/auth/change-password

## 9. 超管重置他人密码
- 入口：用户权限页“重置密码”按钮
- 规则：仅 super_admin 可操作
- root 特殊保护：仅 root 自己可以重置 root 密码
- 接口：POST /api/auth/users/:userId/reset-password
- 入参：new_password（至少 6 位）

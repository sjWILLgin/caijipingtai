# 手工数据采集平台账号与权限矩阵

## 1. 目标
实现最小可用账号体系：
- 注册
- 登录
- 修改密码
- 超管配置用户角色与权限勾选

## 2. 账号体系

### 2.1 用户表
- 表名：sys_user
- 核心字段：username, password_hash, display_name, is_active
- 密码存储：bcrypt 哈希

### 2.2 角色表
- 表名：sys_role
- 角色：
  - super_admin（超级管理员）
  - domain_admin（域管理员）
  - analyst（分析师）

### 2.3 用户角色关联
- 表名：sys_user_role
- 一个用户当前仅配置一个角色

### 2.4 用户权限关联
- 表名：sys_user_permission
- analyst / domain_admin 会自动合并角色默认权限与用户额外勾选权限
- 用户权限页的勾选项用于追加扩展权限，不再用于剥离导入链路基础权限
- super_admin 默认全权限（不走勾选）

## 6. 默认权限策略

### 6.1 super_admin
- 默认拥有全部权限
- 仅 root 可担任该角色
- root 可分配 analyst 与勾选他人权限

### 6.2 analyst（默认）
默认勾选：
- plan.view
- plan.manage
- task.view
- task.create
- task.delete
- file.upload
- mapping.edit
- validation.run
- commit.execute
- commit.rollback
- table.view
- dashboard.view
- audit.view

默认不勾选：
- table.lifecycle
- table.delete
- user.manage
- approval.manage
- meta.manage

说明：
- analyst 页面入口仍与 super_admin 不同，不展示用户权限、信息中心、审批模板、数据维护等系统管理页面。
- analyst 的导入链路执行权限与 super_admin 保持一致，可新建/编辑方案、创建任务、上传、映射、校验、提交、删除任务与回滚批次，避免正常导入流程被接口权限阻断。

### 6.3 domain_admin（域管理员）
默认勾选：
- plan.view
- plan.manage
- task.view
- task.create
- task.delete
- file.upload
- mapping.edit
- validation.run
- commit.execute
- commit.rollback
- table.view
- table.lifecycle

- approval.manage

默认不勾选：

说明：

- domain_admin 的导入链路执行权限与 super_admin 保持一致；审批与表策略能力仍受角色与业务域规则约束。

## 7. API 权限控制说明
系统按接口路径 + HTTP 方法进行权限拦截。
权限判定口径：
- super_admin：直接放行全部权限。
- analyst / domain_admin：接口校验时合并“角色默认权限 + 用户额外勾选权限”后判断。
- 导入链路基础权限属于角色默认权限，不能通过用户权限页取消。

示例：
- POST /api/import-plans 需要 plan.manage；analyst/domain_admin 默认具备该权限
- POST /api/commit/:taskId 需要 commit.execute
- POST /api/commit/batches/:batchId/rollback 需要 commit.rollback
- PUT /api/tables/:tableName/lifecycle 需要 table.lifecycle
- DELETE /api/tables/:tableName 需要 table.delete

## 8. 修改密码流程
- 入口：页面头部“账户下拉菜单 -> 修改密码”
- 校验项：旧密码正确、新密码至少 6 位、确认密码一致
- 接口：POST /api/auth/change-password

## 9. 超管重置他人密码
- 入口：用户权限页“重置密码”按钮
- 规则：仅 super_admin 可操作
- root 特殊保护：仅 root 自己可以重置 root 密码
- 接口：POST /api/auth/users/:userId/reset-password
- 入参：new_password（至少 6 位）

## 10. 超管删除账号
- 入口：用户权限页“删除账号”按钮
- 规则：仅 super_admin 可操作
- 保护规则：root 不可删除，当前登录账号不可自删
- 接口：DELETE /api/auth/users/:userId
- 审计记录：写入 audit_log（log_type=SYSTEM, log_level=WARN, detail.action=DELETE_USER）

## 11. 超管信息中心
- 入口：系统管理 -> 信息中心
- 规则：仅 super_admin 可访问
- 功能：查看指定日期的全员操作日志，支持按操作人、日志类型筛选
- 接口：GET /api/auth/operation-center

## 12. 表级审批流配置与提交校验
- 入口：手工数据表 -> 审批流
- 维护权限：super_admin、domain_admin（域管理员仅可配置其负责域）
- 配置项：是否强制审批、审批角色（super_admin/domain_admin）、指定审批人（可选）
- 当导入提交命中审批条件（方案需要审批或表配置强制审批）时：
  - 自动创建审批单并阻断最终写入
  - 前端提示“审批未完成”并支持刷新审批状态
  - 审批通过后才允许最终写入

## 13. 审批中心与域绑定入口
- 审批中心入口：系统管理 -> 审批中心（super_admin、domain_admin）
- 功能：查看我的待办、我的申请，支持通过/驳回并填写意见
- 域绑定入口：系统管理 -> 用户权限 -> 域绑定（super_admin）
- 功能：为用户维护业务域列表，控制域管理员可处理范围

## 14. 审批模板管理权限口径
- 入口：系统管理 -> 审批流模板
- super_admin：可管理全部业务域的审批模板（新建/编辑/发布）
- domain_admin：仅可管理自己负责业务域的审批模板
- analyst：无审批模板管理权限

## 15. 数据维护（业务域）权限口径
- 入口：系统管理 -> 数据维护 -> 业务域维护
- super_admin：可新增/编辑/启停业务域（维护元仓主数据）
- domain_admin：不可访问数据维护页面
- analyst：不可访问数据维护页面

说明：
- 导入方案等业务页面的“业务域下拉”对已登录用户可读（用于正常业务操作）
- 但业务域主数据写操作仅 super_admin 可执行

模板能力说明：
- 支持串行（SERIAL）、或签（OR_SIGN）、会签（AND_SIGN）
- 支持通过阈值（MIN_PASS_COUNT）与驳回阈值（THRESHOLD_REJECT）
- 支持审批人按用户/角色/域管理员配置
